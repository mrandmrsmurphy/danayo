# Korean lookup problems

Tracking issues found while evaluating whether [[Korean HS]] could be marked `date-last-perfect`. Its own `dataviewjs` datacheck only reports the level of characters *already linked from the page* (`file.outlinks`) — unlike the Stroke/Radical/SKIP lookup pages, it does not run a vault-wide `WHERE hanmun_edu_level = "..."` query, so it cannot catch a missing character on its own. This page records what a manual cross-check (grep every `characters/*.md` with `hanmun_edu_level: 高等`, diff against every character actually linked from `Korean HS.md`) found.
