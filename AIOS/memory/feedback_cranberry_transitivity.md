---
name: feedback-cranberry-transitivity
description: #cranberry marks a shared stand_in only when transitivity holds — both characters mean literally the same thing; cranberry status can also license overriding a character's MC-derived pronunciation
type: feedback
---

Dan'a'yo's blanket definitional rule: a character either stands alone, or has a two-character word meaning *precisely* what it would mean alone — no more, no less. When two different characters share the same `stand_in` compound, that's only legitimate if the two characters' own meanings are identical to each other and to the compound: e.g. 蛞="slug", 蝓="slug", 蛞蝓="slug", so transitively 蛞=蝓. Confirmed cases are tagged **#cranberry** (linguistic term 聯綿詞 — bisyllabic morphemes with no independent meaning apart from the compound, plus dead paired-gender terms like 鳳/凰 or 麒麟 where the modern unified sense now covers both halves).

**Why:** Surfaced while auditing `characters/演.md` and `characters/踊.md` for corrupted `stand_in` fields (both pointed to compounds — 練習, 跳躍 — that didn't even contain the character; an unrelated data-corruption bug, not a transitivity question). That led to reviewing `WTH.md` (root-level scratch file), which lists ~137 character pairs sharing a stand_in. Voting on all 137 (2026-07-12) showed most are *not* cranberries — they're ordinary near-synonym compounds (仇敵: 仇="grudge-foe" vs 敵="rival-foe", distinguishable) that just haven't been given each character its own precise stand-in yet. Only a minority (loanword transliterations like 葡萄/琵琶/菩薩, native alliterative/rhyming binomes like 徘徊/恍惚, dead-sense pairs like 鳳凰/麒麟) actually pass.

One vote had to be reversed: 彷徨 looked transitive at a glance (matches the 徘徊/恍惚 pattern), but 彷's own real meaning is "resemble" — attested by 彷彿, which already exists as a word and already correctly serves as 彿's stand_in. 彷 in 彷徨 is a pure phonetic/rhyming filler (仮借-type sound-loan), not semantically empty — its stand_in was simply wrong (pointed to 彷徨 instead of 彷彿), the same bug class as 演/踊, just disguised as a transitivity question.

**A second consequence, found with 檸檬 (lemon):** cranberry status can license overriding a character's mechanically MC-derived Dan'a'yo pronunciation, when *all three* also hold: (1) `mc_id: 0` (no genuine Classical Chinese attestation, so the MC initial/final are borrowed-by-analogy, not real inheritance), (2) confirmed cranberry (no other usage anywhere), and (3) daughter-language evidence that the word was never actually transmitted as a shared Sino-Xenic reading (e.g. Japanese レモン and Korean 레몬 borrow the Western word directly instead of using on'yomi/Sino-Korean on the characters; Vietnamese uses a native word). Under those three conditions, 檸/檬 were repointed from their MC-derived ㄋ⼶ㄫ/ㄇㄛㄫ to a reading built directly from the international loanword: ㄌㄝ/ㄇㄛㄋ (→ 檸檬 = "lemon"). Contrast with 葡萄/菩薩/琵琶, which are also cranberry but have genuine ancient `mc_id` attestation and real cross-daughter on'yomi correspondence — those should keep their MC-derived readings.

**How to apply:**
- Whenever a `stand_in` is shared by more than one character, don't pattern-match against other cranberry-looking pairs. Check each character individually: does it carry a real, independently-attested meaning elsewhere (another compound, classical usage) distinct from the shared compound's meaning? Yes for either → not transitive; it's either a plain data bug (wrong stand_in, fix it directly) or a genuine near-synonym compound needing its own separate stand-in eventually. No for both → genuinely transitive, tag `#cranberry`. See `WTH.md` for the full per-pair vote log before treating any shared stand_in as already settled.
- Before assuming a cranberry character's MC-derived pronunciation is untouchable, check `mc_id`. If it's `0` and the daughter languages show independent/non-inherited borrowing for the whole word, the pronunciation itself may be a candidate for override — ask the user rather than assuming either way.
