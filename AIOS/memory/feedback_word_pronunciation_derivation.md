When perfecting a word's `зҫ…йҰ¬еӯ—`/`и«әж–Ү`/`жіЁйҹі`, the correct value is a **direct concatenation of each component character's values from `characters/`** вҖ” per `words/BP Words.md`. There is no word-initial devoicing or other sound-shift rule in this conlang.

**Why:** While perfecting `words/иңӮе·Ј.md` and `words/иңӮиңң.md`, I found `и«әж–Ү`/`зҫ…йҰ¬еӯ—` used нҗҒ/pong (г„Ҷ) while `жіЁйҹі` used г„Ҳг„ӣг„« (matching `characters/иңӮ (char).md`'s fong/лҪ•/г„Ҳг„ӣг„«). I incorrectly "fixed" `жіЁйҹі` to match нҗҒ/pong, inventing a sound-shift rule based on `words/иңӮ.md` (which itself has pong/нҗҒ/г„Ҷг„ӣг„« вҖ” inconsistent with the character page). The user corrected me: иңӮ = fong/лҪ•/г„Ҳг„ӣг„«, so иңӮе·Ј = fongjau/лҪ•мһЈ/г„Ҳг„ӣг„«г„җг„ҡг„Ё. I reverted and fixed `и«әж–Ү`/`зҫ…йҰ¬еӯ—` to fongjau/лҪ•мһЈ and fongmid/лҪ•лҜҝ instead.

**How to apply:**
- Always derive a compound's `зҫ…йҰ¬еӯ—`/`и«әж–Ү`/`жіЁйҹі` from `characters/` (the bound/character form), not from another word page вҖ” word pages can themselves be unperfected/wrong.
- If a single-character word page (e.g. `words/иңӮ.md`) disagrees with the corresponding `characters/X (char).md` page, that's a red flag on the *word* page, not evidence of a sound-shift rule. `words/иңӮ.md` (pong/нҗҒ/г„Ҷг„ӣг„«) still disagrees with `characters/иңӮ (char).md` (fong/лҪ•/г„Ҳг„ӣг„«) вҖ” flagged for whenever that word is perfected, but don't change it without checking with the user first.
