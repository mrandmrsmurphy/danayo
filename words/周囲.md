## Notes
- [周](../characters/周.md) "circumference" + [囲](../characters/囲.md) "encircle" — neither character's own `stand_in` points here (周's is 圓周, 囲's is 包囲). `kwin: false` per the AND-rule: `characters/周.md` is `kwin: false`, `characters/囲.md` is `kwin: true` → compound `false`.
- **Real typo corrected**: `cantonese: jau1wai4` — 周 is never read jau1 (its own character page confirms `zau1`); corrected to `zau1 wai4`.
- **Historical-orthography variant moved to prose, not a distinct reading**: `japanese` previously listed `しゅうい,しうゐ` as if two alternatives — しうゐ is not a separate pronunciation but the pre-reform historical kana spelling (歴史的仮名遣い) of the same modern しゅうい reading (verified via search: しう represents the long しゅう sound, ゐ represents modern い). Kept only the modern しゅうい in the field.
- Vietnamese chu vi is a real, extremely common word (the standard math term for "perimeter/circumference"), a clean match.
